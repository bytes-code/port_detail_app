## 3. 数据模型（Model）

```swift
struct PortRecord: Identifiable {
    let id = UUID()
    let pid: String          // 进程 ID
    let processName: String  // 进程名称，如 "python3.11"
    let user: String         // 所属用户
    let protocolType: String // TCP / UDP
    let port: String         // 端口号
    let fullCommand: String  // 完整命令行，如 "python3.11 ./server.py --port 8000"
    let startTime: String    // 进程启动时间，如 "Tue May 12 14:30:00 2026"
}
```

> 命名避免与 Foundation 的 `ProcessInfo` 冲突。`fullCommand` 和 `startTime` 通过 `ps -p PID -o args=,lstart=` 获取。

## 6. 交互流程

```
输入端口号 → 回车/点击查询
    ↓
调用 lsof -i :端口
    ↓
解析结果 → 表格展示
    ↓
用户勾选进程 → 点击 "Kill 选中"
    ↓
弹出确认对话框
    ↓ 确认

    ↓

```

### 状态说明

| 状态 | 表现 |
|---|---|
| 空闲 | 输入框为空，表格显示占位提示"输入端口号开始查询" |
| 加载中 | 查询按钮显示转圈，输入框禁用 |

## 8. lsof 输出解析

`lsof -i :端口` 的典型输出格式：

```
COMMAND   PID USER   FD   TYPE DEVICE SIZE/OFF NODE NAME
node    12345  hc   23u  IPv4 0x...      0t0  TCP *:3000 (LISTEN)
Python  56789 root  5u   IPv6 0x...      0t0  TCP *:3000 (LISTEN)
```

### 解析规则

- 第一行为表头，跳过
- 按空格/空白分割每一行，提取：`COMMAND`、`PID`、`USER`、`NAME`
- 从 `NAME` 字段中提取协议（`TCP`/`UDP`）和端口号（冒号后面的数字）
- 空输出表示端口未被占用
- lsof 的 stderr 输出包含错误（如权限不足），需要捕获
- 拿到 PID 列表后，批量调用 `ps -p PID1,PID2... -o pid=,args=,lstart=` 获取完整命令行和启动时间

```swift
// PortService.swift 核心方法
func queryPort(_ port: String) throws -> [ProcessInfo] {
    let task = Process()
    task.executableURL = URL(fileURLWithPath: "/usr/sbin/lsof")
    task.arguments = ["-i", ":\(port)"]
    // 捕获 stdout 和 stderr
    // 逐行解析，跳过 header 行
    // 返回 ProcessInfo 数组
}

func killProcess(_ pid: String) throws {
    // 优先 kill 不带 -9（SIGTERM），如果进程不响应再 kill -9（SIGKILL）
    let task = Process()
    task.executableURL = URL(fileURLWithPath: "/bin/kill")
    task.arguments = [pid]
}
```

### Kill 策略

- 先尝试 `kill <PID>`（SIGTERM，进程可以优雅退出）
- SIGTERM 失败后再 `kill -9 <PID>`（SIGKILL，强制终止）
- 如果进程属于其他用户，需要提权

## 9. 权限处理

| 场景 | 所需权限 | 处理方式 |
|---|---|---|
| 查询端口（lsof） | 一般不需要特殊权限 | 直接执行 |
| Kill 自己的进程 | 不需要 | 直接 kill |
| Kill 其他用户的进程 | 需要 sudo | 弹出 AppleScript `do shell script with administrator privileges`，系统自动弹出密码输入框 |

```swift
// 提权执行 kill
func killWithPrivileges(_ pid: String) throws {
    let script = "do shell script \"kill -9 \(pid)\" with administrator privileges"
    let task = Process()
    task.executableURL = URL(fileURLWithPath: "/usr/bin/osascript")
    task.arguments = ["-e", script]
}
```

## 10. 输入校验

在 ViewModel 中校验端口号：

| 校验项 | 规则 | 错误提示 |
|---|---|---|
| 非空 | 不能为空 | "请输入端口号" |
| 数字 | 必须为纯数字 | "端口号必须为数字" |
| 范围 | 1 ~ 65535 | "端口号范围 1-65535" |
| 整数 | 不接受小数 | "端口号必须为整数" |

校验失败时不发起查询，直接展示错误提示。
